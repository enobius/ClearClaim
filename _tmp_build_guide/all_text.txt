1. What we are building
An AI multi-agent system with two modes running on the same platform. Patient mode predicts out-of-pocket costs before a visit. Hospital mode predicts claim denial risk before submission — connecting the two sides of healthcare billing that have never been linked.

Gets: cost range, out-of-pocket, insurance split, prior auth flag, Claude explanation, savings tips.

10. Judge Q&A — prepared answers
Is this real data?
"Our cost predictor and NCCI coding validator use real CMS data structures with representative values. Insurance benefits are simulated. The mock layers are clearly separated and swappable — it is one config change."
Can the AI hallucinate a price?
"No. The LLM never generates a number. Every dollar and denial percentage traces to a deterministic lookup against our data files. The LLM only parses input, routes decisions, and writes explanations around real numbers."
Why hasn't this been built?
"Patient cost tools and denial prediction tools exist separately. Nobody has connected them into a single closed-loop system — because it requires understanding both sides of the transaction simultaneously."
What would it take to go to production?
"One key integration: Availity API for real-time benefits verification. The pricing layer already uses CMS-structured data. The LangGraph architecture is designed so each mock layer is a single swappable module."
Why Claude instead of GPT-4o?
"Claude excels at careful, structured reasoning on clinical text — exactly what denial risk analysis and documentation gap detection require. Claude also follows complex structured output instructions more reliably, which matters when agents depend on parseable responses from each other."
